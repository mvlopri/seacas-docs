36 /*!
37  * This set of classes provides a thin wrapper around the exodusII
38  * internals. It supplants several of the exodusII API calls in
39  * order to avoid ncredef calls which totally rewrite the existing
40  * database and can be very expensive. These routines provide all
41  * required variable, dimension, and attribute definitions to the
42  * underlying netcdf file with only a single ncredef call.
43  *
44  * To use the application must create an Internals instance
45  * and call the Internals::write_meta_data() function. This
46  * function requires several classes as arguments including:
47  * <ul>
48  * <li> Mesh -- defines mesh global metadata
49  * <li> Block -- defines metadata for each block
50  * <li> NodeSet -- defines metadata for each nodeset
51  * <li> SideSet -- defines metadata for each sideset
52  * <li> CommunicationMetaData -- global metadata relating to
53  * parallel info.
54  * </ul>
55  *
56  * Calling Internals::write_meta_data(), replaces the
57  * following exodusII and nemesis API calls:
58  * <ul>
59  * <li> ex_put_init(),
60  * <li> ex_put_elem_block(),
61  * <li> ex_put_node_set_param(),
62  * <li> ex_put_side_set_param(),
63  * <li> ne_put_init_info(),
64  * <li> ne_put_loadbal_param(),
65  * <li> ne_put_cmap_params(),
66  * </ul>
67  */
