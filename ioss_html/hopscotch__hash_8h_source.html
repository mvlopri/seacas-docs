145  /*
146  * Each bucket may store up to three elements:
147  * - An aligned storage to store a value_type object with placement-new.
148  * - An (optional) hash of the value in the bucket.
149  * - An unsigned integer of type neighborhood_bitmap used to tell us which
150  * buckets in the neighborhood of the current bucket contain a value with a hash
151  * belonging to the current bucket.
152  *
153  * For a bucket 'bct', a bit 'i' (counting from 0 and from the least significant
154  * bit to the most significant) set to 1 means that the bucket 'bct + i'
155  * contains a value with a hash belonging to bucket 'bct'. The bits used for
156  * that, start from the third least significant bit. The two least significant
157  * bits are reserved:
158  * - The least significant bit is set to 1 if there is a value in the bucket
159  * storage.
160  * - The second least significant bit is set to 1 if there is an overflow. More
161  * than NeighborhoodSize values give the same hash, all overflow values are
162  * stored in the m_overflow_elements list of the map.
163  *
164  * Details regarding hopscotch hashing an its implementation can be found here:
165  * https://tessil.github.io/2016/08/29/hopscotch-hashing.html
166  */
167  static const std::size_t NB_RESERVED_BITS_IN_NEIGHBORHOOD = 2;
